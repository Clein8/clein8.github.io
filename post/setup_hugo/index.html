Github Pages에 Hugo 설치하기

❏ Github Pages

언제부턴가 구글에서 코드 검색을 하다 보면 .github.io 사이트를 자주 볼 수 있었다. 잠깐 찾아보니 Github Pages라길래, Github에서 블로그 같은 서비스를 만들었나 싶었다.

그러던 도중 .github.io 사이트들에서 보이는 Code Highlighter가 Tistory에 설치하는 Code Highlighter보다 예뻐 보이고, 코드 입력하기도 더 쉬워 보이길래 좀 더 관심을 가지게 되었다. 며칠을 검색해 보니 코드 저장 해놓기에는 Tistory보다 좋다고 생각되어 시작해 보기로 했다.

그리고, 다 설치하고보니 마크다운(Markdown)으로 글 작성하는게 생각보다 편하고 좋다. ㅎㅎ



❏ 정적 블로그 (Static Site Generator) - Jekyll, Hugo, Hexo …

일단, 정적 블로그 중 제일 유명한 건 Jekyll이다. 오래되고 사용자도 그만큼 많은 것 같다. 하지만 검색해 보니 Post 숫자가 많아지면 느려진다는 이야기가 있길래. 홈페이지 Loading이 느리다는 건 줄 알았는데.. PC에서 사이트 생성이 오래걸린다는 소리였다. 정적 블로그 개념이 없어서 오해를 했었다.

사실, 정적 블로그라는 명칭 보다는 영문명 그대로 정적 사이트 생성기라고 보는게 맞다.. Theme를 선택하고 마크다운 문법으로 문서를 작성하면 사이트 생성기가 사이트를 만들어 준다. 웹서버 없이 그냥 html, css, js 파일 등으로 생성되기 때문에. 그냥 업로드 공간만 있으면 된다. 그래서 Github Pages에 사이트를 올릴 수 있는 것이다. 어떻게 보면 대단하고 다른 한편으로는 무식한거 같다.

이틀 동안 Hexo 설치 -> Hugo 설치 -> Jekyll 설치 -> Hugo 설치를 해 봤다.

처음에는 각자 장단점이 뭔지 인터넷에서 아무리 읽어봤자 뭘 골라야 할지 몰라서 골치아프고 그랬는데, 셋 다 깔아 봤더니 답이 나왔다(처음에는 셋 다 깔아볼 생각은 전혀 없었음). 잠깐 사용해 본 거라 각자의 장단점을 모두 파악 할 순 없었지만, 어쨋는 Hugo로 결정. 밑에 비교 사이트가 있긴 하지만, 개인적으로 별 도움이 안됬다.

stackshare 비교 (어제는 Jekyll이 왼쪽이었는데 오늘은 Hugo가 왼쪽이다.) 둘이 자리 싸움 하는거 같다. Jekyll이랑 Hugo랑 서로 기준 한다고 싸우나 보다. 다시 Jekyll이 왼쪽이다.

Hugo를 사용하기로 했으면 이제 설치를 하자. 설치 글인데 지금까지는 잡담이었다. 이제 드디어 설치 시작. ㅎㅎ 다른게 더 좋다면.. bye bye.

Windows랑 Mac에 둘 다 설치해 봤는데. 그냥 사이트에서 운영체제에 맞는 파일 받아서 설치하면 된다. 간단하다.

3. Hugo 사이트 생성 및 설정

ps. 이틀간 구글신께 의뢰해 받은 사용법이며, 다른 방법도 검색 되었으나 저는 이렇게 사용한다는 예시입니다.

3-1. Hugo 사이트 생성

$ hugo new site {hugo_site_name}

ex)
$ hugo new site hugo

위에 처럼 입력해 주시면 아래 처럼 축하한다는 메시지와 함께, {hugo_site_name}에 입력한 이름의 폴더가 생성 된다.

Congratulations! Your new Hugo site is created in /Users/{user_name}/{hugo_site_name}.

Just a few more steps and you're ready to go:

1. Download a theme into the same-named folder.
   Choose a theme from https://themes.gohugo.io/, or
   create your own with the "hugo new theme <THEMENAME>" command.
2. Perhaps you want to add some content. You can add single files
   with "hugo new <SECTIONNAME>/<FILENAME>.<FORMAT>".
3. Start the built-in live server via "hugo server".

Visit https://gohugo.io/ for quickstart guide and full documentation.


3-2. 테마 추가

Hugo Themes에 들어가서 마음에 드는 테마를 선택 한 후, Homepage를 누르면 해당 테마의 Repository로 이동합니다. Repository에서 git 주소를 복사해서 {theme_url}에 입력하면 된다. 

$ cd themes
$ git clone {theme_url}
$ cd ..
cf. 현재 사용중인 테마는 Mainroad라는 테마 입니다.


3-3. Hugo 기본 설정 (config.toml)

Hugo사이트를 처음 생성하고 config.toml파일을 열어보면 다음과 같이 나온다. 

1baseURL = "http://example.org/"
2languageCode = "en-us"
3title = "My New Hugo Site"
다른 사이트 생성기에도 비슷한 화면이 보인다. Jekyll보다 Hugo가 더 간단한거 같다. (벌써 Jekyll config 내용이 기억이 잘 안난다. ㅠㅠ)

다음과 같이 설정하면 된다. Build할 때 Theme를 따로 지정 할 수도 있지만 여기에 테마를 적어 놓으면 기본 Build시 config 파일 안에 적혀 있는 Theme를 사용한다. 

1baseURL = "https://clein8.github.io/"
2languageCode = "ko-kr"
3title = "Clein's Repository"
4theme = "Mainroad"
5
6googleAnalytics = ""
Hugo Syntax Highlighting <- Highlighter 설정 참조 (이건 더 파서 글을 새로 작성해야 겠다.)

나머지 설정은 Theme테마 제작자 마다 설정 방법이 달라서 테마 제작자가 작성해 놓은 설정방법을 참조.


3-4. Hugo 서버 localhost 실행

$ hugo server -D

이 서버는 Hugo 사이트가 잘 만들어 졌는지 확인하고, 마크다운으로 글 작성하면서 브라우저에 어떻게 적용하는지 볼 수 있는지 확인할 수 있게 해주는 localhost 서버이다.

-D는 비공개 글까지 다 보여 주라는 옵션이다. 글 작성시 draft: true로 되어 있으면 사이트에 비공개 처리 되는데. draft 설정 무시하고 다 보여주는 옵션이다. 확인용으로 사용하기 때문에 -D 붙이는게 편하다.. 습관적으로 붙이도록 하자. (다른 블로그로 설명하자면 관리자 로그인 했을때 비공개 글 까지 보이는 거라고 생각하면 쉽다.)

접속은 http://localhost:1313, 종료는 Ctrl+C

따로 Build는 하지 않고, 이미 Build된 파일만 보여주는 용도로 사용되는거 같다. (서버 도는 중에, 카테고리 또는 태그를 새로 추가 해도 서버 중지 하고 Build다시 할 때 까지 해당 링크가 404 뜬다.)


3-5. Hugo Git 연결

Jekyll은 저장소 하나로 연동이 가능하다고 하는거 같다. 안타깝게도 Hugo는 전체를 올릴려면은 2개의 저장소가 필요하다. 하나는 Hugo file용 다른 하나는 Build된 public 폴더 안에 파일 용이다. .github.io에는 Build된 public 폴더 안에 든 파일을 올려야 한다. 처음에는 public 폴더 용만 올릴려고 생각 했었는데, 다른 컴퓨터에서 글을 쓰거나 나중에 컴퓨터를 바꿀 일을 대비해서. 그냥 둘 다 올려 두기로 했다.

일단 {hugo_site_name} 폴더 전체를 새로운 Repository와 연결을 한다. 

$ git remote add origin https://github.com/{user_id}/{repository_name}.git

$ git submodule add https://github.com/{user_id}/{user_id}.github.io.git public

submodule 이라는게 하위 repository 같은 느낌인데, index.html 파일이 public 폴더 안에 생겨서 hugo 파일은 따로 보관하고.. public 폴더만 .github.io에 연결해서 사이트 기능을 하게 하는거 같다.

사이트만 올릴 려면 public 폴더에서 아래 하나만 실행하면 된다. 

$ git remote add origin https://github.com/{user_id}/{user_id}.github.io.git


3-6. Hugo Build

$ hugo

or

$ hugo -t {theme_name}
간단하다. hugo만 입력하면 된다. -t 옵션으로 다른 테마 선택 가능하다. 하지만 딱히 그럴 이유도 없고config.toml 파일에 기본 Theme를 지정해 놨기 때문에 그냥 Build하면 된다.

public 폴더에 생긴 파일들을 Github Pages에 올려 주면 된다.


3-7. 새 글 작성

$ hugo new post/test.md
위 처럼 입력하면 /content/post/test.md파일이 생성된다. 아래 처럼 작성해 주면 된다.

---
title: "test"
date: 2019-03-29T16:49:59+09:00
draft: true
---
여기에다 마크다운으로 본문 입력

post는 기본 글 설정이 되어 있는 폴더이다. post 폴더에 글을 쓰면 index page에 알아서 추가된다. 아래 처럼 특정 페이지나 따로 폴더를 입력해도 된다. 

ex)
$ hugo new about.md
$ hugo new books/book-name.md


3-8. Github에 Push

$ hugo
$ cd public
$ git add .
$ git commit -m "blah blah"
$ git push origin master
$ cd ..
$ git add .
$ git commit -m "blah blah"
$ git push origin master

push하기 전에 Build는 습관적으로 해주자 ㅎㅎ

public 폴더 먼저 push 하고. hugo 폴더 나중에 Push해야 한다.. 아마 submodule 때문인거 같다.

올릴때마다 이 짓을 해야 하는게 너무 불편해서 검색을 해 보니 쉘 스크립트를 만드는 방법이 있었다. 쉘 스크립트 말고 다른 방법도 있는거 같은데. 이것으로 만족하도록 한다.
