정적 사이트 생성기 (Static Site Generator) 비교/선택

인터넷 검색하다 보면 사람들이 ****.github.io라는 주소로 블로깅을 하는걸 보고, 처음에는 Github에서 블로그 서비스를 시작한줄 알았다.

Tistory에서 Code Highlighter사용하는게 너무 불편해서, 나도 ****.github.io를 사용해 볼까 하고 검색을 하다 보니 정적 블로그라는 것을 알게 되었다.

정적 사이트 생성기를 이용해서 블로그를 만드는 경우가 많다 보니 정적 블로그라고 불리게 된 것 같다. 그러나 몇개의 사이트 생성기를 사용해 보니 영어를 직역한 표현이긴 해도 정적 사이트 생성기라는 이름이 알맞은 이름 같다.


❏ 정적 사이트 생성기 종류

일단, 선두주자로 제일 오래된 Jekyll이 있다. 검색하면 제일 많이 나오고, 만들어진 사이트도 많다.

그 다음으로 Hugo와 Hexo를 알게 되었는데, 처음에는 이 셋중에 어느걸 골라야 하는지 인터넷에서 검색도 해보고 stackshare 비교 등도 찾아보고 했지만, 뭐가 뭔지 하나도 모르겠어서 셋 다 설치해 보았다. 삽질의 결과 위 셋 중에는 Hugo가 제일 마음에 들어서. Hugo로 정착하게 됬다.

그러던 도중 VuePress와 Gatsby라는 것도 알게 되었다. Hugo보다 좋아 보이길래 갈아탈 요량으로 설치를 해 봤는데, 개인적으로 VuePress가 제일 깔끔해 보이고 마음에 들었지만 VuePress는 블로그 보다는 기술 문서 등에 사용하기에 좋아 보이고, 굳이 블로그를 만들수는 있긴 한거 같은데.. 블로그에는 그냥 Hugo가 더 낫다는 결론이 나왔다.

다음은 각 사이트 생성기들의 개발 언어들인데, 딱히 알 필요는 없는거 같다.. 아직 오래 사용해 보진 않았지만, 처음 사이트 생성할 때 명령어 몇줄 빼고는 쓸 일이 없는거 같다. (뭔가 더 비교해서 채워 넣을려고 했는데 귀찮다.)
